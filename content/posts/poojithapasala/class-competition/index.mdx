## Task summary

The task is to determine if two given spans of text were written by the same author. This is a binary classification problem where the documents are labeled as 0 for "not the same author" and 1 for "same author."

## Exploratory data analysis

1.The training CSV file consists of three columns: ID, TEXT and  LABEL, while the test CSV file contains only the ID and TEXT columns. It was observed that the TEXT field included special characters, which required the use of UTF-8 encoding for proper handling.

2.Missing values in the TEXT column of both the training and test files were addressed by replacing them with empty strings. It was found that no missing values were present in the TEXT field of either dataset.

3.The distribution of text span lengths and labels (same author vs. different authors) was analyzed to evaluate potential class imbalance within the dataset.

4.The diversity of the dataset is analyzed by stylometric features such as average word length, unique word and character diversity and vocabulary diversity , which helped in understanding the variation in writing styles across the classes.

## Project Approach

### Importing Libraries & Loading Data: 
- Imported necessary libraries for data handling (pandas, numpy), text processing (re, nltk), model training (torch, transformers) and evaluation metrics (sklearn.metrics).
- Downloaded and configured additional resources, such as NLTK stopwords, to ensure proper text preprocessing.

### Data Loading and Preprocessing:
1. Loaded Train and Test Data:
- Read train.csv and test.csv using pandas with UTF-8 encoding to handle potential encoding issues.
- Handled missing values in the TEXT column by filling them with empty strings, ensuring no data was lost during preprocessing.
2. Preprocessed Text Data:
- Defined a custom preprocessing function that cleaned the text while retaining stylistic elements important for authorship profiling.
- Applied this function to the TEXT column of both the train and test datasets, ensuring consistent preprocessing for model training and evaluation.

### Feature Extraction:
1. Used BERT Tokenizer:
- Initialized a BERT tokenizer (bert-base-uncased) with a maximum sequence length of 256 tokens, preparing the text for BERT-based feature extraction.
2. Applied TF-IDF Vectorizer:
- Extracted additional text features using a TfidfVectorizer with a maximum of 50,000 features and n-gram range (1, 3) to capture both unigram and bigram patterns.
Fitted and transformed the train dataset and the test dataset to ensure consistent feature representation.
3. Computed Class Weights:
- Used class_weight.compute_class_weight to handle class imbalance in the dataset, ensuring that underrepresented classes were given more weight during training.

### Model Architecture:
1. Loaded Model Components:
- Loaded a pre-trained BERT model (bert-base-uncased) for feature extraction to capture semantic information from the text.
- Defined a linear layer for TF-IDF features to reduce dimensionality (128 dimensions), ensuring efficient integration of these features with BERT.
- Combined the BERT features and TF-IDF features into a single classifier to make predictions.
2. Created Custom Model Wrapper:
- Created a wrapper class CustomModelWrapper to integrate BERT, TF-IDF and the classification head into a single PyTorch module, making the model easy to manage.
- Defined a forward pass to compute combined features from both BERT and TF-IDF inputs and make predictions for authorship classification.

### Implemented Evaluation Metrics:
- Implemented compute_metrics to calculate Accuracy(to measure overall performance),
Precision, recall, F1-score (to evaluate the model’s performance in handling imbalanced classes and its ability to distinguish between authors).
- Used these metrics for model evaluation during training to guide optimization and hyperparameter tuning.

### Trained with K-Fold Cross-Validation:
- Split the training data into 3 folds using StratifiedKFold to ensure balanced class distribution.
- Configured training parameters such as output directory, batch size, number of epochs and metric for saving the best model (F1-score).
- For each fold, prepared the datasets, initialized a Trainer object, trained the model and evaluated it on the validation set while logging metrics.
- Aggregated the mean F1-score across all folds to evaluate the overall model performance.

### Generated Test Predictions and Submission:
- Tokenized the test dataset and included TF-IDF features to ensure consistency in feature representation during prediction.
- Used the trained model to predict labels for the test set, classifying authorship based on the input text.
- Created a DataFrame containing ID and LABEL columns to format the results according to submission requirements.
- Saved the results as submission.csv for final evaluation.

### Motivation:
The motivation behind this project is to build a reliable model that can identify the author of a text by combining two powerful techniques: BERT embeddings for capturing the meaning of words and TF-IDF features for capturing the style of writing. By using K-fold cross-validation, the model is trained and tested on different data splits, ensuring that it performs well on unseen data. The goal is to create an effective and accurate classifier for authorship attribution, even when dealing with class imbalances in the dataset.

### Challenges:
- One of the key challenges faced during the development of this model was the prolonged runtime for each epoch, particularly when training with a larger number of epochs. The kernel frequently failed when the model was tested with five epochs, likely due to resource constraints. To address this, I had to reduce the number of folds in the cross-validation process to three, which helped alleviate computational load. Additionally, to improve performance and ensure stability, I limited the size of the linear classifier to 128 dimensions, striking a balance between model complexity and computational efficiency.

### Discussion of SotA approaches:
- SotA approaches in authorship attribution leveraged pre-trained language models like BERT, which capture deep contextual information from text. Combining BERT embeddings with traditional text features such as TF-IDF allows the model to not only understand the content but also the stylistic elements of the writing. Additionally, K-fold cross-validation is widely used for robust model evaluation, ensuring that the model generalizes well across different subsets of data. This hybrid approach balances modern deep learning techniques with classical methods to achieve accurate authorship classification.

## Results
- The F1 score achieved on the Kaggle leaderboard is 0.57150, representing an improvement over the F1 score of 0.51626 obtained using logistic regression, which was used earlier.
- Using stratified k-fold cross-validation ensured fair representation of class distributions across folds, crucial for evaluating an imbalanced dataset. For instance, the model's F1 score improved consistently across folds: 0.706 in fold 1, 0.789 in fold 2 and 0.941 in fold 3, highlighting its ability to generalize better with varied subsets of data. Early epochs showed low precision due to undefined predictions, which improved as the model learned from balanced splits. The approach reduced overfitting risks, provided robust metrics (e.g., precision improving from 0.706 in fold 1 to 0.941 in fold 3) and offered a reliable estimate of the model's performance across different data distributions.
## Error analysis

1. **Class Imbalance Impact**: The significant class imbalance in the dataset (1245 instances of class 0 vs. 356 instances of class 1) may lead to bias in the model, causing it to favor the majority class (label 0). This could result in underperformance when predicting the minority class (label 1). This imbalance may also contribute to instability in model performance across different folds.

2. **Underfitting in Fold 1**: In Fold 1, the model's evaluation loss of 0.6645 indicates potential underfitting or instability. The moderate accuracy (70.79%) and F1-score (70.70%) suggest that the model is not yet learning the patterns well, potentially due to a lack of capacity or inadequate training, which needs addressing.

3. **Fluctuating Metrics in Fold 2**: Although performance improves in Fold 2 (with an evaluation loss of 0.6436 and accuracy of 79.59%), there is some instability in the third epoch, where precision and recall metrics slightly fluctuate. This instability indicates that the model's predictions are inconsistent, potentially due to overfitting or insufficient generalization during training.

4. **Challenges with Data Augmentation**: Despite training on a larger dataset with data augmentation, the model did not show significant improvement in performance, suggesting that the augmentation strategy might not have been effective. This could be due to the augmentation not addressing the underlying class imbalance or the augmented data not adding meaningful diversity for the model to learn from.
