## Project description

### Introduction
Our project aims to evaluate open-source GPT-style models—specifically GPT-2, GPT-Neo, and GPT-J—for the task of generating code snippets from natural language descriptions in real-time. This endeavor seeks to aid developers by providing immediate code suggestions, enhancing productivity and efficiency in programming tasks.

### Goals
- Determine the most efficient and accurate model for real-time code generation.
- Understand the trade-offs between model complexity and performance.
- Provide insights into the practicality of using open-source models in development tools.

### Statistical NLP Aspect
We will examine how the statistical properties of language models affect their ability to generate syntactically correct and functional code in real-time applications. This includes analyzing factors like model architecture, training data distribution, and decoding strategies.

### Project Scope
The project is appropriately scoped for the course, focusing on fine-tuning existing models and evaluating their performance within a manageable timeframe and resource availability.

### Rough Timeline
#### Week 1 (Nov 15 - Nov 21):
- Model Selection: Choose GPT-2, GPT-Neo, and GPT-J for comparison.
- Dataset Preparation: Collect and preprocess a dataset of natural language descriptions and corresponding code snippets.
#### Week 2 (Nov 22 - Nov 28):
- Model Fine-Tuning: Fine-tune each model on the prepared dataset.
- Benchmarking: Establish baseline performance metrics using standard evaluation datasets.
#### Week 3 (Nov 29 - Dec 5):
- Performance Evaluation:
  - Assess models based on accuracy (code correctness), latency (response time), and resource utilization (memory and compute requirements).
  - Real-Time Testing: Deploy models in a simulated real-time environment to evaluate practical performance.
  - Analysis & Comparison: Analyze the strengths and weaknesses of each model.
  - Reporting: Compile findings into a comprehensive report with recommendations for future work.
