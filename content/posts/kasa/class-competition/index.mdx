## Task summary

- In this task, we are required to determine if two spans of text were written by the same author. 
- This task falls under author profiling and authorship identification. 
- The dataset is a collection of text pairs joined by a [SNIPPET] delimiter, and the objective is to classify these pairs as either "same author" (1) or "not the same author" (0).
- The training data is provided in CSV format, with a test set for final evaluation. Performance is measured using the macro F1 score to ensure balanced precision and recall for both classes.

## Exploratory Data Analysis

#### Dataset Overview

- The dataset contains two partitions: a training set (`train.csv`) and a test set (`test.csv`). 
- Each document contains a pair of text spans joined by the `[SNIPPET]` delimiter. 
- Our target is binary (0 or 1), indicating whether both spans were authored by the same individual.
- The dataset consists of a training set and a test set. 
- Each entry contains two text spans, with a binary label indicating whether the spans were authored by the same individual.

#### Key observations

- Text span lengths vary significantly, ranging from short sentences to entire paragraphs.
- The class distribution is imbalanced: the majority of the labels in the dataset are "not the same author."
- The dataset includes variations in punctuation, capitalization, and stylistic choices that could signal different authorship, making it important to account for these features during preprocessing.
#### Additional Analysis
- I computed various stylometric features, such as average sentence length, word length, lexical diversity, readability score, and counts of key punctuation marks. 
- These features are important for distinguishing between writing styles.

## Approach
#### Preprocessing:

- I cleaned the text by removing punctuation, digits, and extra whitespace. 
- I converted the text to lowercase for consistency.
#### Feature Engineering:

- I applied TF-IDF vectorization (up to trigrams) to capture the frequency-based importance of words in the text spans. 
- The vocabulary size was capped at 10,000 to balance performance and computational efficiency.
- I engineered stylometric features, including average sentence length, word length, lexical diversity, readability score, and punctuation counts (e.g., commas, periods, exclamation marks). 
- These features capture writing style characteristics, which are crucial for distinguishing authorship.
#### Model:

- I combined TF-IDF features with stylometric features and trained a neural network model with two hidden layers, each followed by dropout for regularization.
- I used logistic regression with balanced class weights to address class imbalance.
- After testing multiple configurations, I selected a neural network with 128, 64, and 32 neurons across layers, using ReLU activations and a sigmoid output for binary classification.
#### Evaluation:

- I employed the macro F1 score to evaluate model performance, ensuring balanced precision and recall between both classes.
- Novelty: In addition to standard TF-IDF features, incorporating stylometric features like lexical diversity and readability added richness to the model’s understanding of writing style, which improved classification accuracy.
## Results

- The model achieved an F1 score of 0.53442 on the training set. 
- To validate the robustness of the approach, I used train-test splits and cross-validation to monitor performance and reduce overfitting.

## Error analysis

- The model struggled with certain text pairs where stylistic differences were subtle. 

For example:
- Texts with similar sentence structures but different punctuation usage often led to misclassifications.
- False Positives: The model occasionally classified two text spans as being written by the same author when stylistic features like sentence length and punctuation counts were similar, despite differences in content or tone.
- False Negatives: Shorter text spans led to challenges in identifying authorship, especially when limited vocabulary or sentence structure information was available.
- In future iterations, I plan to further investigate the model's errors by focusing on such borderline cases and potentially incorporating more advanced linguistic features like syntactic structure or sentiment.
