## Task Summary

The **task description** focuses on a binary classification problem in the domain of authorship identification, a subfield of [author profiling](https://en.wikipedia.org/wiki/Author_profiling) and digital stylometry. The objective is to train a model to predict whether two given spans of text share the same author. The task involves pairwise comparison of text spans from a labeled training dataset and applying the model to unseen data in an unlabeled test dataset (`test.csv`).

The **task design** is structured to evaluate a model's ability to capture stylistic features and infer authorship similarity from the text spans. This formulation emphasizes the challenge of pairwise comparison and highlights the role of textual stylistic features in solving the problem. The evaluation metric for this task is the macro F1 score, designed to assess the model's accuracy and recall performance across both classes.

The **label space** consists of two distinct categorical classes: `0` and `1`, with no missing values. These classes are defined as follows:

- `0`: The pair of text spans were authored by different individuals.
- `1`: The pair of text spans were authored by the same individual.

The **dataset size** and class distributions are summarized in the table below. The training data exhibits class imbalance, with approximately 78% of examples labeled as `0`. Estimates for the test set and total dataset assume a similar class imbalance, with an expected overall distribution near 75% for class `0` and 25% for class `1`.

### Related Tasks
Authorship identification shares similarities with several related tasks in natural language processing (NLP), including:

- Authorship Verification: Determining if a text was authored by a specific individual, often using one reference text.
- Plagiarism Detection: Identifying similarities between documents to infer potential copying or paraphrasing.
- Textual Style Transfer: Modifying text to emulate the writing style of another author, which involves capturing stylistic features similar to those required for authorship identification.

### Challenges of the Task
Some of the challenges of this task include:

- Class Imbalance: The dataset's skewed distribution, with significantly more examples labeled as 0, poses a challenge for balanced model performance.
- Stylistic Variability: Text spans vary greatly in length and linguistic complexity, making it difficult to extract consistent stylistic features.
- Domain Diversity: Text sources, primarily from Project Gutenberg, may introduce noise or non-representative patterns due to variations in era, genre, and content.
- Ambiguity in Authorship: Some texts may exhibit overlapping stylistic features, leading to potential confusion for both models and human annotators.

### State of the Art (SotA) Approaches
Three SotA approaches relevant to this task include:

1. Stylometric Analysis: Using handcrafted linguistic features (e.g., word frequencies, sentence structure) to represent authorial style. Tools like Principal Component Analysis (PCA) can be applied for feature reduction.
2. Neural Approaches: Transformer-based models like BERT fine-tuned for pairwise text classification. These models capture deep contextual relationships between text spans.
3. Ensemble Models: Combining classical machine learning (e.g., SVMs trained on stylistic features) with neural methods to leverage both handcrafted and learned representations for robust performance.

## Exploratory Data Analysis

The **dataset design** is tailored for the task. It is sourced from works in the public domain. There works are primarily from [Project Gutenberg](https://www.gutenberg.org/) eBooks and enriched by additional aggregated text sources, processed for task relevance. The **dataset size** for this task consists of 5,000 English-language text spans in 2,500 pairs. **Per class*, approximately 75% (~1,875) of the text pairs were not written by the same person (`LABEL` = 0) and the other 25% (~625) were (`LABEL = 1`).

The text spans vary significantly in length, from brief excerpts of a few dozen characters to lengthy passages of a few thousand. Text spans are separated by a delimiter token, `[SNIPPET]`, marking transitions between the first and second span in each `TEXT` cell. See the descriptions of individual dataset files for more details.

### Observations

#### Text Length Variability
The length of text spans varies significantly, even within the same pair. This poses challenges for models, as longer spans offer more stylistic features, while shorter spans often lack distinctive tokens, reducing their utility for authorship detection.
- **Extreme Example**:
  - **ID 1802**: Span 1 contains only the letter `E`, while Span 2 has 677 characters with no `E` or relevant entities.
- **Other Examples**:
  - **ID 624**: Includes a 34-character snippet ("When Lilacs Last in the Dooryard B") easily identifiable as Walt Whitman.
  - **ID 971**: Contains a 59-character generic snippet ("he asked. 'Of course.' And I did, too...") with limited stylistic markers.

#### Incomplete Text Capture
Issues with text preprocessing impact the dataset’s usability:
- Missing sentence beginnings or endings reduce context (e.g., ToDo, ToDo).
- Incomplete quotation marks hinder interpretation of dialogue (e.g., ToDo, ToDo).
- Line-feed removal merges paragraphs, erasing natural breaks (e.g., ToDo, ToDo).

#### Named Entity Overlap
Approximately 8-9 percent of span pairs (e.g., IDs 22, 44, 2489, 2499) contain overlapping named entities, which is a moderately strong indicator (~0.65) of shared authorship. However, common entities like "one" or "first" can introduce bias, leading to false positives.

### Additional Analyses

#### Measures of Diversity
- **Token Diversity (Vocabulary)**:
  Vocabulary diversity, calculated as the ratio of unique tokens to total tokens, is higher in longer spans but limited in shorter spans, further contributing to imbalance in stylistic richness across pairs.
- **Named Entity Diversity**:
  Diversity in named entities varies across spans, with longer spans typically containing more entities. Short spans often rely on generic entities, limiting their utility for authorship prediction.

#### Jaccard Similarity
- **Named Entity Similarity**: Jaccard similarity between spans’ named entities highlights overlap in ~8% of pairs, serving as a useful feature for predicting shared authorship.
- **Vocabulary Overlap**: Jaccard similarity of vocabulary across spans is higher for pairs with `LABEL=1`, suggesting shared authorship is more likely when linguistic overlap exists.

#### TF-IDF Analysis
Token-level analysis using Term Frequency-Inverse Document Frequency (TF-IDF) reveals distinctive features across spans. High-weight tokens often correspond to unique named entities, proper nouns, or distinctive lexical choices. For example:
- ID 624 emphasizes "Lilacs" and "Dooryard" as key tokens linked to Walt Whitman.
- ID 971 lacks strong TF-IDF tokens, reflecting its generic composition.

## Approach

Based on my initial understanding of the competition rules, I planned to purse three different approaches to the authorship detection task. I aimed to explore a variety of methods to maximize my chances of success and to learn more about the dataset and the task. My three planned approaches were:

  1. **Google Search API**: Use the Google Search API to search for the text spans and extract the author's Project Gutenberg (PG) book ID number (BID) and Creator ID number (CID). I would then use these IDs to determine if the text spans were written by the same author.

  2. **Stylometric Analysis**: Use stylometric analysis to extract stylistic features from the text spans, such as word frequencies, sentence structure, and named entities. I would then use these features to train a machine learning model to predict authorship similarity.

  3. **Deep Learning**: Implement a deep learning model, such as a neural network network or a Transformer model, to learn the stylistic features of the text spans and predict authorship similarity.

In the end, the first approach produced near perfect results, but was outside the competition rules. For the third approach, I was unable to develop a model that exceeded the weighted-random baseline. Only, the second approach using just named-entity overlap as a stylometric feature produced legal results that exceeded the weighted-random baseline.

### Approach 2 - Named-Entity Overlap
My final approach was simply to set all labels to `0`, since the super-majority of text pairs were written by different authors. Then, I looked for features that would support a change of label to `1`. The first feature I engineered was named-entity overlap, since the vast majority of texts were excerpts from novels and short-stories. I used spaCy to produce a document object for each span and extracted the overlapping set of named entity tokens between the text pairs into a new dataframe column, `df['neo']`. I then calculated the Jaccard similarity in a new `df['neo_sim']` column to quantify the degree of overlap. Several, common overlapping entities introduced false positives compared to the training data, so I manually created a list of common entities, `excluded_neos`, to remove from the overlap sets and similarity calculations. The code then set the label to `1` if `df['neo']` was not an empty set and `df['neo_sim']` was greater than a certain value.

My motivation for selecting this approach was frustration with my poor performance from deep learning methods and a looming deadline. I decided to try a simple approach that would be easy to implement and could potentially yield results better than weighted-random baseline. The named-entity overlap approach was quick to implement and provided a reasonable baseline for comparison with more complex models. While it did not achieve the highest F1 score, it was a valuable learning experience and helped me better understand the dataset and the task at hand.

## Results

Results from the 'NEO' approach moderately exceeded the weighted-random baseline. The second and final submission (`submission_ner02.csv`) scored **0.59065**--4th place on the leaderboard. This final score is a 16.0% **delta** over the weighted-random baseline of 0.50901. In addition, it is only 7.6% below the top score of 0.63940.

## Error analysis

Based on **detailed error analysis** of 20% **held-out data**, the **characterization of errors** included both false positives and false negatives.

  - **False Positives** were primarily in rows with token overlap on common entities (e.g., 'one', 'first'). For example, the two text spans in ID 315 share the entity 'the United States'. The spans in ID 1461 share the name 'George'. Neither pairs of text were authored by the same individual.
  - **False Negatives**
    - Short-spans. These rows lacked sufficient token overlap on which to infer authorship similarity (e.g., ID 1194-1).
    - Non-fiction works also caused false negatives (e.g., ID 421, 576). Since the vast majority of the dataset was from works of fiction, the model was biased towards predicting shared authorship for these pairs.
