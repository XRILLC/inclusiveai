## Task summary

- The task involves predicting whether two text spans were authored by the same individual. The dataset contains **1601 training entries** and **899 test entries**, with a binary label space:
  - **Class 0**: Text spans from different authors (**1,245 samples**)
  - **Class 1**: Text spans from the same author (**356 samples**)

The task involves text preprocessing (lemmatization and stopword removal), feature extraction using TF-IDF and sentence embeddings, and similarity metrics (cosine, Euclidean, Manhattan). A stacked ensemble model combining XGBoost and Random Forest is used to predict the labels.

**Task challenges**
- **Class Imbalance**: Class 0 dominates (78%), leading to potential bias in predictions and lower accuracy for Class 1.
  - Example: Models may overpredict "different authors" (Class 0) due to its prevalence and fail to predict "same authors" (Class 1) as much.
- **Writing Style Variability**: Differences in syntax, vocabulary, and tone within a single author or across authors can obscure patterns.
  - Example: An author’s formal vs. casual texts could appear to be from different writers.
- **Feature Overlap**: Similarities in writing style across authors can lead to false positives.
  - Example: Two different authors using formal academic language may confuse the model. -

## Approach
-1. **Data Loading and Initial Inspection**:
   - Loaded the training and test datasets (`train.csv` and `test.csv`).
   - Inspected the structure and key columns (`TEXT`, `LABEL`) to understand the data format and target variable.

2. **Text Splitting**:
   - Split the `TEXT` column into `span1` and `span2` using the `[SNIPPET]` delimiter to isolate the two text spans for similarity comparison.

3. **Text Preprocessing (Data Cleaning)**:
   - Applied text cleaning using SpaCy, including lowercasing, lemmatization, and removal of stopwords and non-alphanumeric tokens.

4. **Exploratory Data Analysis (EDA)**:
   - Analyzed sentence lengths, vocabulary diversity, and class distribution.
   - Visualized sentence length distributions and identified class imbalances.

5. **Feature Extraction**:
   - Used **TF-IDF Vectorization** to convert text data into numerical features (unigrams and bigrams).
   - Applied **Sentence-BERT embeddings** to capture the semantic meaning of sentences.
   - Calculated **cosine similarity**, **Euclidean distance**, and **Manhattan distance** between the spans.

6. **Handling Class Imbalance**:
   - Applied **class weighting** or **oversampling** to handle the imbalanced dataset.

7. **Model Selection and Hyperparameter Tuning**:
   - Chose **XGBoost** for its efficiency and performance with high-dimensional data.
   - Used **Halving Random Search CV** for hyperparameter optimization.
   - Implemented a **Stacking Classifier** combining XGBoost and **Random Forest** for improved accuracy.

8. **Training and Model Evaluation**:
   - Split the data into training and validation sets.
   - Evaluated the model using accuracy to assess performance and generalization.

9. **Final Predictions**:
   - Applied the trained model to the test data and generated predictions.
   - Saved the results in a CSV file for submission.

**Motivation behind using this approach**:
The approach was designed to systematically preprocess, extract relevant features, and address challenges like class imbalance to improve model performance.
Text preprocessing and feature extraction focused on both lexical and semantic features to better capture text similarities.
By utilizing advanced methods like **BERT embeddings** and **XGBoost**, combined with ensemble learning and hyperparameter tuning,
the goal was to build a robust and high-performing model for the text classification task.

## Key Features and Innovative Aspects
1. **Combination of Traditional and Modern Techniques**:
   - Blends **TF-IDF** (for capturing term importance) with **Sentence-BERT embeddings** (for semantic understanding), enhancing feature extraction by leveraging both lexical and semantic features.

2. **Multiple Similarity Metrics**:
   - Utilizes **cosine similarity**, **Euclidean distance**, and **Manhattan distance** to provide a comprehensive view of text similarity, capturing various aspects of relationships between the spans.

3. **Stacking Classifier**:
   - Implements a **Stacking Classifier** combining **XGBoost** and **Random Forest**, leveraging the strengths of both models for improved performance and generalization.

4. **Efficient Hyperparameter Tuning**:
   - Incorporates **Halving Random Search CV** to speed up hyperparameter optimization while ensuring effective model tuning, making the search process more efficient.
