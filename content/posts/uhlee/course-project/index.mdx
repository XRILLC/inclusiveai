## Project description

This project aims to compare several open-source generative models' capabilities in text-summarization, using song lyrics from The Beatles. From there, the summaries would be evaluated against human-written summaries. The metrics for evaluation include accuracy on the theme, coherence, and relevance. The human-written summaries were written by a third-party, Frank Uhle. Frank is a published author, film and music historian, established Beatles fan, and my father. 

My motivation for this project was the curiosity about the differences between the capabilities between LLMs. I am also a lifelong Beatles fan, having been introduced to them as a young child by my dad. Therefore this project was an exciting opportunity to combine these two interests.

A total of five songs were initially selected for this project, chosen for their lyrical complexity, story-telling, and differing themes.

## Results

Each model was given the same instructions for each prompt: "Please summarize the following song in 3 sentences," followed by the name and artist of the song. I included the results in their entirety for each model's response. I ranked each response in the following categories: truthfulness (was the summary accurate about the song?), conciseness (was the summary too verbose?), instruction-following (was the summary 3 sentences long?), and style (were there any spelling or grammatical errors?). Each category was ranked from 1 to 10, with 10 being the best and the final results are the average of scores for each song.

## Error analysis

Potential errors in this project could be due to finding lyrics online that may have a few incorrect words, or misspellings that lead to ambiguity in the meaning. I also used a feature in AnvilGPT that allows you to simultaneously see the response from each model side by side. The order was llama3.2:latest, gemma:latest, and tinyllama:latest. Tinyllama consistently scored worst of the 3 models. It may be possible that if I interacted with each model individually that the results could have been different.

## Reproducibility

Reproducibility of this project is difficult due to the varied outputs of LLMs. Putting the same prompt into the same model multiple times in a row will almost always yield a different response. However, if one was to try to recreate this experiment on their own, the following would need to be done. The song lyrics were all sourced from publicly available information on the internet, and the models used were open-source LLMs that are free to use.

## Future improvements

Future improvements to this project could include providing a more specific and detailed prompt to the LLM, such as "Please summarize this song in 3 sentences, _specifying the theme, and main characters in the song._" for example (i.e. giving futher explicit instructions for the model to follow). Other improvements could be selecting several other human summaries, perhaps from people that are unfamiliar with the Beatles song, who may have never heard the song and must review only the lyrics (without music) much the same way that the model sees the song. I selected my dad as the human-summarizer due to his familiarity with the Beatles, and his history as a writer. He is also college-educated, with a Master's in Library Science and adept at following instructions. This probably skewed the results in favor of the human summary as well. Additionally, the rankings were done only by myself, therefore the project would benefit from multiple evaluators.
