## Task summary
The task involves examining the linguistic features of two texts to determine if they were written by the same author. For our binary classification model, I utilized a dataset where text pairs are separated by a [SNIPPET] delimiter. While most of the data comes from Project Gutenberg, I supplemented it with additional collected and preprocessed sources. The dataset comprises approximately 1,601 entries, with 356 samples marked as "1" (same author) and 1,245 samples marked as "0" (different authors).Deep Learning Models: Transformer-based models like BERT or RoBERTa are often applied in authorship attribution tasks due to their ability to capture contextual semantics. However, their computational requirements and sensitivity to class imbalance make them less ideal for smaller datasets.
Class Imbalance: The training dataset had a significant imbalance, with "different author" samples outnumbering "same author" samples by over threefold, leading to bias in model predictions. For instance, this imbalance caused baseline models like Logistic Regression to predict the majority class inaccurately.
Text Variability: Variability in text length, style, and punctuation made feature extraction challenging. For example, dialogues in one text span but not the other introduced misleading stylistic signals.
Data Augmentation Techniques: Methods such as back translation and paraphrasing can generate additional training samples, addressing imbalance issues effectively and improving model robustness without significant computational overhead.

## Approach

Upon conducting exploratory analysis, it became clear that the training dataset was not only relatively small but also highly imbalanced, which presented a challenge for accurate model performance.
The initial step involved building a baseline model using LogisticRegression, acknowledging that the class imbalance could hinder its effectiveness. As expected, this model achieved only 42% accuracy on the imbalanced test set, largely due to the over representation of class "0" in both training and testing datasets. This highlighted the need for a new strategy to better handle the imbalance.

**Data Splitting**
The training data was partitioned into training and test sets using a 75/25 split to ensure effective model evaluation.

Feature Engineering
To represent features, I applied the TfidfVectorizer on preprocessed training texts. The preprocessing steps included converting text to lowercase, removing non-alphanumeric characters, and eliminating stopwords prior to vectorization.

ML Model Training and Evaluation Metrics
I trained a Random Forest Classifier with hyperparameters optimized through RandomizedSearchCV. The model was trained on the training set and evaluated on the test set, achieving a final accuracy of 53.65%. The classification report, detailed in the results section, provides metrics such as precision, recall, and F1-score for each class.

## Error analysis

The model achieves a respectable overall accuracy of 71%, but a closer examination of precision and recall metrics for each class reveals critical insights into performance gaps and areas for improvement. The model struggles particularly with the minority positive class (class 1) compared to the negative class (class 0).Initially for class 1, precision is 69% and recall is 82%, whereas class 0 shows 69% precision but only 52% recall. The low recall for class 0 indicates that the model fails to correctly identify over half of the actual negative samples, often misclassifying them as false positives. Many of these false positives arise from challenges such as confusing dialogues or unique character names that appear only in specific text spans, which skew the stylistic similarity signals used for authorship attribution.

Additionally, the model's difficulty in accurately classifying the underrepresented positive class highlights an imbalance issue. It tends to default to predicting different authorship, leading to the observed low recall for class 0. Addressing this imbalance could involve targeted augmentation of same-author examples to create a more evenly distributed training dataset.

Key issues identified from error analysis:

Misleading stylistic signals due to dialogues or unique character names, necessitating more robust feature extraction methods.
Training data imbalance, which biases the model toward predicting different authorship, requiring balanced augmentation strategies.
