## Task summary
The task involves examining the linguistic features of two texts to determine if they were written by the same author. For our binary classification model, we utilized a dataset where text pairs are separated by a [SNIPPET] delimiter. While most of the data comes from Project Gutenberg, we supplemented it with additional collected and preprocessed sources. The dataset comprises approximately 1,601 entries, with 481 samples marked as "1" (same author) and 1,120 samples marked as "0" (different authors).

b- Key Observations and Challenges:
The training data shows a clear class imbalance, with 1,120 pairs labeled as "0" (different authors) compared to just 481 labeled as "1" (same author).
We analyzed simple syntactic features, such as punctuation and conjunction usage, across text spans in both classes. However, relying solely on punctuation counts can be misleading.
There is considerable variability in the length of text spans, both across different datasets and within individual pairs in the same dataset, making it challenging to use absolute length features as reliable indicators.
Dialogues frequently appear in one text span but not in the other, complicating the use of straightforward word count metrics for classification.

## Exploratory data analysis

Approach

Upon conducting exploratory analysis, it became clear that the training dataset was not only relatively small but also highly imbalanced, which presented a challenge for accurate model performance.
The initial step involved building a baseline model using LogisticRegression, acknowledging that the class imbalance could hinder its effectiveness. As expected, this model achieved only 42% accuracy on the imbalanced test set, largely due to the over representation of class "0" in both training and testing datasets. This highlighted the need for a new strategy to better handle the imbalance.
-In my second attempt to address the imbalance, I hypothesized that reducing the over represented class "0" samples would help equalize the class distribution. I applied undersampling to reduce the number of "0" samples to match the count of "1" samples. However, this approach resulted in a dataset that was too small to effectively train the model. As a result, a RandomForest model trained on this undersampled data achieved only 46% accuracy, which was still below the initial baseline.
-Realizing that more samples labeled as "1" were required, I employed web scraping to collect over 100 English books, extracting more than 1,000 additional text pairs with the "1" label. This enriched dataset was then combined with the complete set of original "1" samples and a subset of undersampled "0" samples, resulting in a more balanced and larger training dataset. Training a RandomForest model on this enriched data led to a validation accuracy of 46.5%, demonstrating the value of augmenting data to improve class representation.

-To further enhance the model’s performance, I performed hyperparameter tuning on the RandomForest model using GridSearch. After testing different combinations of hyperparameters, the optimized model achieved a test accuracy of 48%, marking the best performance obtained during the entire process. ## Results

The RandomForest model demonstrated its effectiveness by achieving an accuracy of 68% on the held-out test set. To further evaluate the model's robustness, I employed 5-fold stratified cross-validation on the entire training dataset, optimizing for F1-score. The cross-validation yielded an average accuracy of 63% with a low standard deviation of ±2%, indicating that the model consistently performs well across various data splits.
