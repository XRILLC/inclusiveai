## Task summary

Full disclosure, I just realized the day that the assignment is due that I completely misunderstood what this assignment was asking. I thought that each cell of the data was one document and we were trying to compare them to see if that match the author which would get the label 1 and the other docs would receive 0 denoting it was not the same author. 
I did not do any data analysis until today which made me realize my mistake. Fully understand if I fail this assignment but wanted to submit at least something. The tast summary written below was before I realized my mistake.

I was originally going to use word2vec in my implementation for this project. Unfortunately, I was never able to get the score above .49 so I ended up using a BERT model as it already has pre-training to help boost my accuracy.
I hoped that using the BERT to provide the logit scores would be more accurate as there are so many factors to consider when trying to differentiate text between authors. 
The way my model works, the dataset is run through tokenization before any processing happens. The BERT model receives training from the training data to help tailor it to this specific task. Then I the test data is passed through the BERT model to receive logit scores as embeddings.
After the test data is processed and the embeddings are ready, the logistic regression model takes the logit stores and provides the final label assignments. I also have training loss, validation loss, and accuracy set up so that I could monitor loss as I changed various settings. A portion of the training data was held aside to be used for the loss tracking during training.
A big challenge I ran into with this was processing time, it takes about an hour an half for this to complete. When first trying to use the BERT model, it actually wouldn't run due to my computer having not enough processing power, so I had to limit the settings to a small learning rate and with only 3 training epochs.
The long processing time lead to a small issue as last week, I got the model to a point where my score was .55, but I forgot to save what settings I had for that run, and since then have tried to get a better score. 
As you can imagine, my model is now performing poorly, and I am unsure why and I don't have the settings from my highest leaderboard post to revert back to. 
Revisiting the discussion of my orginal idea of using word2vec, I was hoping that it would be enough to capture enough of the similarities and differences in writing styles between authors. One of the challenges of this task is that recognizing writing styles boils down to more than just what words are used. There are sentiments, points of view, content choices, etc..
Unfortunately, not matter how robust I tried to make my word2vec embeddings, I was never able to break the .5 score barrier.

## Exploratory data analysis

One thing I noticed that could potentially cause some confusion for a model would be conversations. In document 1065, the first snippet contained a conversation between two characters while the second snippet was describing a situation and contained only a little conversation.
If one of the snippets only contained a scene with dialogue while the other contained a description or though process of a scene or character, but it was indeed the same author, the LLM could misinterpret the inclusion or absence of dialogue as a feature indicative of a certain author. 
To shift to how this data could make classification easier would be document id 959. It is clear that the first snippet is set in a different time period and most likely a fictional universe due to this excerpt, "sandcar caravans out in Syrtis." The second snippot mentions flights and autopilots which contrasts heavily with the first snippet.
Although upon further consideration, I suppose this could technically be the same author (it isn't) if the author had branched out and written different stories. This is part of what makes this task difficult. Writers can have the same writing style, but it can be difficult to differentiate if they change genres and topics completely. 
The easiest ones that could be considered would be if writing from someone like Shakespeare is compared to a modern author.
In documents such as id 583, its easier to tell that it's the same author as it is very dialogue heavy and one could change out the character names to be matching between the two and it would sound like the same book at different points.

## Error analysis

For error analysis, as I have mentioned, I held out 20% of the training data in order to track training loss, validation loss, and accuracy. I used those number as my gauge for how my changes in setting were effecting the performance of my model. At first, if my validation loss increased from the first epoch, I would shift the settings again and re-run the cell. 
I quickly realized that this model continuously had an increase in validation loss every time so I ended up just letting the model finish, and I had to upload the results as a prediction to truly gauge how the model was doing. I realize now that this is likely due again to the fact that some of the documents had more than one author. 
Having the different writing styles being considered as one most likely caused my model to struggle throughout the label assignment task.
