Based on the wikipedia page on author profiling there are various techniques that can be applied to predict information about the author, which in turn can help with identifying an author from text.
Usual process:
- Identifying specific features to be extracted from text
- Building an adopted, standard representation (Bag of Words model) for the target profile
- Building a classification model using a standard classifier (SVM) for the target profile.

Problems:
- Wide variation in lengths of texts used
- Class imbalance in data

So ML models are SVMs; Naive Bayes Classifiers; Deep averaging layers in a cycle of machine learning that uses the mean of word embeddings within a text; also long short-term memory.
- for Blogs Instance based learning and random decision forests worked. 

Other possible implementations include 
Feature Engineering: identify the set of distinguishable characteristics which uniquely describe each author class.

    - Apply ML algo
    - Evaluate results

## Approach

_See [the rubric](https://parsertongue.org/courses/snlp-2/assignments/rubrics/class-competition/final/#approach)_

Data Processing:
I removed stopwords from the data, and segmented the TEXT column into sentences.
Then I pulled out the features from the data for the Naive Bayes model, and the tokenized the data via word embeddings for the other models.

Model implementation:
First began with Naive Bayes, Random Forest and BERT. Naive Bayes was the most accurate of all 3 models.
Then I moved to CNN. This created the most accurate predictions.

## Results
